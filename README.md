# 超市收银促销系统 (Java 面向对象实现)

本项目为面试题：模拟超市计价系统，支持多种水果组合及多重促销策略叠加。

## 1. 题目需求
### 基础规则
- **苹果**：8 元/斤
- **草莓**：13 元/斤
- **芒果**：20 元/斤

### 演进场景
1. **顾客 A**：购买苹果、草莓，计算总价。
2. **顾客 B**：增加芒果，计算总价。
3. **顾客 C**：在 B 的基础上，草莓限时 **8 折**。
4. **顾客 D**：在 C 的基础上，购物 **满 100 减 10**。

---

## 2. 技术设计思路
为了应对题目中不断增加的水果种类和促销规则，本项目采用了以下设计模式：

- **策略模式 (Strategy Pattern)**：将促销逻辑（打折、满减）抽象为独立策略，支持灵活组合和链式调用，符合“开闭原则”。
- **高精度计算**：全流程使用 `BigDecimal` 处理金额，避免 `double` 浮点数运算产生的精度丢失问题。
- **面向对象模型**：
    - `Fruit`: 封装商品属性。
    - `ShoppingCart`: 封装购买行为。
    - `PromotionStrategy`: 抽象促销接口。

---

## 3. 环境要求
- JDK 1.8+
- IDE: IntelliJ IDEA / Eclipse

## 4. 如何验证结果
运行 `src/com/supermarket/pay/Main.java` 中的 `main` 方法，程序将自动模拟 A、B、C、D 四个场景并输出计算结果，验证逻辑正确性。
